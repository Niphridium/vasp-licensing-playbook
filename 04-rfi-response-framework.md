# 4. RFI response framework

After you apply, the regulator sends Requests for Information (RFIs). How you
handle them often decides your timeline. Sloppy, slow or evasive answers cost
months; precise, evidenced ones build trust.

## Principles
- **One owner, one tracker.** Every RFI item logged with status, owner, due date.
- **Answer the question asked** — not the one you wish they'd asked.
- **Evidence, not assertions.** "We screen daily" → attach the policy + a sample.
- **Map to the rule.** Reference the specific rulebook clause you're satisfying.
- **Be honest about gaps.** "Not yet in place; here's the dated remediation plan"
  beats overclaiming, which destroys credibility if later disproven.

## Response structure (per item)
1. **Restate** the request in one line.
2. **Answer** directly and concisely.
3. **Evidence** — list the attached documents / references.
4. **Rule mapping** — the clause this satisfies.
5. **Remediation** (if partial) — owner + date.

## Red flags to never trigger
- Inconsistencies between documents (business plan vs. application vs. investor deck).
- Undisclosed UBOs or nominee arrangements.
- A business model that, on its face, describes sanctions or AML circumvention.

Any one of these can sink an application — keep the whole document set consistent
and truthful.
